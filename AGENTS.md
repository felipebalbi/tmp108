# AGENTS.md

Canonical guidance for AI coding agents (and humans) working on the
`tmp108` crate. Read this file first. The file at
`.github/copilot-instructions.md` is intentionally minimal and defers
to this document.

## What this crate is

`tmp108` is a `#![no_std]`, platform-agnostic Rust driver for the
Texas Instruments [TMP108] I┬▓C temperature sensor, built on the
[`embedded-hal`] (1.0) and optionally [`embedded-hal-async`] traits.
The crate is part of the [Open Device Partnership][ODP].

- **Target audience:** firmware authors using `embedded-hal`-compatible
  MCUs (Cortex-M, RISC-V, ŌĆ”) that need to read temperature from a
  TMP108 over I┬▓C, either blocking or async.
- **Non-goals:** this crate does not own the I┬▓C bus (it takes an
  `I2c` by value), does not provide a logger, does not pull in `std`,
  and does not provide bus-sharing primitives (use
  [`embedded-hal-bus`] for that).

The register layout, field encodings and accessor methods are
*generated* at compile time from `tmp108.toml` by the
[`device-driver`] crate (toolkit mode). See `src/inner.rs` for the
generated module ŌĆö **do not hand-edit it**.

[TMP108]: https://www.ti.com/lit/gpn/tmp108
[`embedded-hal`]: https://docs.rs/embedded-hal
[`embedded-hal-async`]: https://docs.rs/embedded-hal-async
[`embedded-hal-bus`]: https://docs.rs/embedded-hal-bus
[`device-driver`]: https://docs.rs/device-driver
[ODP]: https://github.com/OpenDevicePartnership

## Building and testing

All commands are run from the repo root. CI uses `--locked` for every
build/check; do the same locally so you exercise the pinned graph in
`Cargo.lock`. Every command below has been run on this branch and
exits 0.

### Formatting (`fmt` job in `check.yml`)

`rustfmt.toml` sets `group_imports` and `imports_granularity`, which
are **nightly-only** options. Stable `cargo fmt --check` will print
warnings but still succeed; CI runs it on nightly. Prefer nightly
locally when you can:

```powershell
# Stable (works, emits two warnings about unstable options ŌĆö acceptable)
cargo fmt --check

# Nightly (matches CI exactly)
cargo +nightly fmt --check
```

### Clippy (`clippy` job)

CI runs this exact invocation on `stable` and `beta`:

```powershell
cargo clippy --all-features --all-targets --locked -- `
    -W clippy::suspicious -W clippy::correctness -W clippy::perf -W clippy::style
```

Note: the lints are passed as `-W` on the command line, but
`Cargo.toml` already promotes `correctness`, `suspicious`, `perf`,
`style`, and `pedantic` to `deny` for the crate itself (see
`[lints.clippy]`). New code must be clippy-clean at those levels.

### Tests (`test` job)

CI runs three matrix invocations:

```powershell
cargo test --locked
cargo test --locked -F async
cargo test --locked -F async,embedded-sensors-hal-async
```

All unit tests live inline in `src/lib.rs` (see the `mod tests` block
around `src/lib.rs:684`). They drive the driver through
[`embedded-hal-mock`].

### Docs (`doc` job)

```powershell
cargo doc --no-deps --all-features --locked
# CI sets RUSTDOCFLAGS="--cfg docsrs" and runs this on nightly
```

There is currently **one pre-existing rustdoc warning** about a
broken intra-doc link (`Configuration`) in `src/lib.rs`. Treat
it as known background noise ŌĆö do not introduce new rustdoc
warnings.

### Feature powerset (`hack` job)

```powershell
cargo install cargo-hack
cargo hack --feature-powerset check --locked
```

This catches feature-additivity bugs. Any time you add or modify a
`[features]` entry, run this.

### MSRV (`msrv` job)

`rust-version = "1.90.0"` in `Cargo.toml`. CI runs `cargo check
--locked` against `1.90`:

```powershell
rustup toolchain install 1.90
cargo +1.90 check --locked
```

### no-std cross-builds (`nostd.yml`)

```powershell
rustup target add thumbv6m-none-eabi thumbv7em-none-eabihf thumbv8m.main-none-eabihf
cargo check --target thumbv6m-none-eabi      --no-default-features --locked
cargo check --target thumbv7em-none-eabihf   --no-default-features --locked
cargo check --target thumbv8m.main-none-eabihf --no-default-features --locked
```

`--no-default-features` is critical: the default build pulls in
`embedded-hal` only, but the dev/example deps (e.g. `tokio`,
`pico-de-gallo-hal`) are host-only.

### Dependency policy (`deny` job)

```powershell
cargo install cargo-deny
cargo deny --all-features --locked check
```

See `deny.toml` for allowed licenses (MIT, Apache-2.0, Unicode-3.0)
and current advisory exemptions.

### Semver (`semver` job)

```powershell
cargo install cargo-semver-checks
cargo semver-checks
```

Treat any reported breakage as PR-blocking unless a major version
bump is intended (see `Cargo.toml:3`).

## Code conventions

### Line endings

- No `.gitattributes` and no `.editorconfig`. All tracked files are
  stored as **LF** in the git index (`git ls-files --eol` shows
  `i/lf` for every file). New files **must be LF**. If you are on
  Windows, set `git config core.autocrlf false` in this repo (the
  agent workflow does this automatically).

### Formatting

- `rustfmt.toml`:
  - `max_width = 120`
  - `imports_granularity = "Module"` (nightly)
  - `group_imports = "StdExternalCrate"` (nightly)
- Run `cargo +nightly fmt` before sending a PR to get the import
  grouping right.

### Lints / safety

`Cargo.toml` enforces (crate-wide):

```toml
[lints.rust]
unsafe_code = "deny"
missing_docs = "deny"

[lints.clippy]
correctness = "deny"
suspicious  = "deny"
perf        = "deny"
style       = "deny"
pedantic    = "deny"
```

- **No `unsafe`.** This is a pure-safe driver. Do not add `unsafe`
  blocks; if you genuinely need one, open an issue first.
- **All public items must have doc comments.** `missing_docs` is
  `deny`. Use `///` doc comments; describe `# Errors` for any
  function returning `Result` (see existing `read_configuration`,
  `temperature`, `configure` in `src/lib.rs`).
- The generated `inner` module is opted out via `#[allow(...)]`
  attributes at its `mod inner;` declaration in `src/lib.rs:27-32`.
  Don't remove those ŌĆö `device-driver` output is not clippy-clean and
  doesn't carry doc comments on every item.

### `no_std` rules

- The crate is `#![cfg_attr(not(test), no_std)]` (see
  `src/lib.rs:13`). Tests run with `std` available; library code
  must not use `std::` paths. Use `core::` and (where needed)
  `alloc::` ŌĆö there is currently no `alloc` dependency.
- Do not add `std`-only dependencies without gating them behind a
  feature.

### Error handling

- Driver methods return `Result<T, I2C::Error>` so the user keeps
  the bus's native error type. Do not wrap or remap I┬▓C errors.
- The `# Errors` rustdoc section is required on `pub` fallible
  functions (clippy `pedantic` enforces this).

### Async / blocking split

- The blocking and async APIs share one source file via
  [`maybe-async-cfg`]. Methods are annotated with
  `#[maybe_async_cfg::maybe(...)]` and the bodies switch on
  `#[cfg(feature = "async")]` for the actual await call (see
  `src/lib.rs:203-261`).
- When you add a new method:
  1. Annotate the `impl` block (and the type if needed) with
     `#[maybe_async_cfg::maybe(sync(...), async(...))]`.
  2. Inside the method, use `#[cfg(feature = "async")]` / `#[cfg(not(...))]`
     to call `*_async().await` vs the blocking variant from
     `device-driver`.
  3. Mirror the method signature exactly between branches.
- The `async` feature flag toggles the entire surface ŌĆö there is no
  "both at once" build. Tests are run in both modes by CI.

[`maybe-async-cfg`]: https://docs.rs/maybe-async-cfg

### Logging

- No `defmt`, no `log`, no `println!` in `src/`. The driver is
  silent by design. If you need diagnostics, return an error or use
  `read_all_registers` (generated helper in `src/inner.rs`) at the
  call site.

### Feature flags

Defined in `Cargo.toml:30-33`:

| Feature                       | Pulls in                          | Notes |
|-------------------------------|-----------------------------------|-------|
| (default)                     | `device-driver`, `embedded-hal`   | Blocking API |
| `async`                       | `embedded-hal-async`              | Swaps the entire public surface to async |
| `embedded-sensors-hal`        | `embedded-sensors-hal`            | `TemperatureSensor` trait impl |
| `embedded-sensors-hal-async`  | `embedded-sensors-hal-async`      | Async sensor traits + `AlertTmp108` (see `src/lib.rs:152-201`); only meaningful together with `async` |

All features must remain **additive**. Verify with `cargo hack
--feature-powerset check`.

## Driver / HAL specifics

### Register source of truth

`tmp108.toml` (root) is the [`device-driver` toolkit][dd-toolkit] TOML
description of the chip's four registers: `temperature` (0,
read-only), `configuration` (1, RW, reset `0x1022`), `t-low` (2,
RW), `t-high` (3, RW). Field encodings (e.g. `ConversionMode`,
`Hysteresis`, `Polarity`, `Thermostat`) are generated as Rust enums.

[dd-toolkit]: https://docs.rs/device-driver

### Generated code (`src/inner.rs`)

`src/inner.rs` is the cached, generated output of
`device-driver`'s build process. **Do not edit it by hand.** If you
need to change a register layout or field encoding:

1. Edit `tmp108.toml`.
2. Re-generate `src/inner.rs`. The repo currently checks in the
   generated file rather than running the macro at build time, so
   regenerate using the same `device-driver` version pinned in
   `Cargo.toml` (`1.0.9`) and commit the diff together with the TOML
   change.
3. Run `cargo build --all-features` and the full test suite.

The generated module is `mod inner;` in `src/lib.rs:27-32`, gated
with `#[allow(clippy::all)]`, `#[allow(clippy::pedantic)]`,
`#[allow(missing_docs)]`, `#[allow(unsafe_code)]`, `#[allow(unused)]`.

### Adding a new public method

Pattern to follow (see `Tmp108::temperature` at `src/lib.rs:268`):

```rust
#[maybe_async_cfg::maybe(
    sync(
        cfg(not(feature = "async")),
        self = "Tmp108",
        idents(AsyncI2c(sync = "I2c"), AsyncDelayNs(sync = "DelayNs"))
    ),
    async(feature = "async", keep_self)
)]
impl<I2C: AsyncI2c> Tmp108<I2C> {
    /// One-line summary.
    ///
    /// # Errors
    ///
    /// `I2C::Error` when the I┬▓C transaction fails.
    pub async fn my_method(&mut self) -> Result<T, I2C::Error> {
        #[cfg(feature = "async")]
        let res = self.inner.<register>().read_async().await;
        #[cfg(not(feature = "async"))]
        let res = self.inner.<register>().read();
        // ŌĆ” convert raw to T ŌĆ”
    }
}
```

Then add tests in both `tests::blocking` and `tests::r#async`
modules at the bottom of `src/lib.rs`.

### `AlertTmp108`

`AlertTmp108` (`src/lib.rs:152-201`) is the variant that owns an
ALERT pin in addition to the I┬▓C bus. It is gated on
`all(feature = "embedded-sensors-hal-async", feature = "async")`.
Keep the gating identical on any new alert-related code; otherwise
the feature-powerset check will fail.

### I┬▓C addressing

The four legal 7-bit addresses (`0x48..=0x4b`) are encoded in the
`A0` enum and selected by the constructors `new_with_a0_{gnd,vplus,sda,scl}`
(`src/lib.rs:108-130`). Do not expose a raw-address constructor.

## Commit & PR conventions

From `.github/copilot-instructions.md` and `CONTRIBUTING.md`,
cross-checked against `git log --pretty=%s`:

- **Subject:** capitalised, Ōēż50 chars, imperative mood (e.g.
  `Bump device-driver to 1.0.9`, `Fix config register byte order`).
  GitHub auto-appends `(#N)` on merge.
- **Blank line, then body wrapped at 72 chars** explaining *what*
  and *why*.
- **Each commit must build cleanly** (`CONTRIBUTING.md` ┬¦"Clean
  Commit History"). Squash typo fixes before pushing.
- **No squash-merge.** The maintainers keep a linear, meaningful
  history.
- **Draft PRs first.** Make sure all CI passes before requesting
  review.
- **DCO / sign-off:** humans add `Signed-off-by:`. AI agents
  **must not** add `Signed-off-by:`.
- **AI-attribution trailer (mandatory for AI-assisted commits):**

  ```
  Assisted-by: AGENT_NAME:MODEL_VERSION [TOOL1] [TOOL2]
  ```

  Examples:
  - `Assisted-by: GitHub Copilot:claude-opus-4.7`
  - `Assisted-by: GitHub Copilot:gpt-5 coccinelle`

  Don't list ordinary editor / VCS / cargo tooling. Agents must
  verify their own model name before composing the trailer ŌĆö
  no copy-pasting from a previous session.

### Branch naming

Feature/topic branches use kebab-case (`add-example-pico`,
`improve-agentic-workflow`, ŌĆ”). Branches are pushed to the
contributor's fork; PRs target `OpenDevicePartnership/tmp108:main`.

## What not to do

- **Don't edit `src/inner.rs` by hand.** Edit `tmp108.toml` and
  regenerate.
- **Don't add `unsafe`.** `unsafe_code = "deny"` at the crate level.
- **Don't add `std::` paths in `src/`.** The crate is `no_std` for
  every target except `cfg(test)`.
- **Don't introduce new clippy warnings at `correctness | suspicious | perf | style | pedantic` level.** They are denied.
- **Don't drop or modify `--locked`** in any CI invocation. The
  pinned `Cargo.lock` is part of the contract.
- **Don't add a raw-`u8` address constructor for `Tmp108`.** Use
  the `A0` enum.
- **Don't add `defmt` / `log` / `println!`** in library code.
- **Don't push to `OpenDevicePartnership/tmp108` directly.** Work
  on a fork and open a PR.
- **Don't add `Signed-off-by:` from an AI agent.**
- **Don't force-push** to shared branches.

# Copilot Instructions

## Commit Messages
- Subject line: capitalized, 50 characters or less, imperative mood (e.g., "Fix bug" not "Fixed bug")
- Separate subject from body with a blank line
- Wrap body text at 72 characters
- Use the body to explain *what* and *why*, not *how*

## AI Attribution
Every commit that includes AI-generated or AI-assisted work **must** contain an `Assisted-by` trailer in the commit message:
```
Assisted-by: AGENT_NAME:MODEL_VERSION [TOOL1] [TOOL2]
```
Where:
- `AGENT_NAME` is the name of the AI tool or framework (e.g., `GitHub Copilot`)
- `MODEL_VERSION` is the specific model version used (e.g., `claude-opus-4.6`)
- `[TOOL1] [TOOL2]` are optional specialized analysis tools used (e.g., `coccinelle`, `sparse`, `smatch`, `clang-tidy`)
Basic development tools (git, cargo, editors) should not be listed.
AI agents **must** verify their own identity (agent name and model version) before composing the `Assisted-by` trailer ŌĆö do not assume or hard-code a model name from a previous session.
AI agents **MUST NOT** add `Signed-off-by` tags. Only humans can certify the Developer Certificate of Origin.
